Setting Up a Human-Readable BOLT12 Offer

Following Seth for Privacy's Guide on Bitcoin Usernames

April 23, 2025

Summary of Findings

This document outlines the process and key observations from setting up a human-readable BOLT12 Lightning Network payment offer as described in Seth for Privacy's guide. This approach uses BIP353 to connect a domain name with a Lightning payment option.

Key Steps Completed

  1. Enabled Core Lightning Experimental Features to support BOLT12 offers
  2. Generated a BOLT12 Offer using the Core Lightning CLI
  3. Created a DNS TXT Record in the format specified by BIP353
  4. Verified DNS Setup using appropriate tools
  5. Tested the Bitcoin Username with compatible wallets

Key Observations

  • BOLT12 offers provide a more user-friendly approach to Lightning payments
  • The DNS-based username system works well for human-readable payment addresses
  • Implementation requires Core Lightning with experimental features enabled
  • Only compatible with certain wallets that support BOLT12 and BIP353

Conclusion

The setup of a human-readable BOLT12 offer was successful, providing a more convenient way for others to send Bitcoin payments. The technical implementation follows the BIP353 specification and leverages DNS for username resolution, creating a balance between decentralization and usability.
